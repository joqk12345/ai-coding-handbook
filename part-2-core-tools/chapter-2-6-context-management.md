# 第2.6章：实战中上下文的管理与添加

在与 Claude 协作编程时，上下文管理至关重要。你的项目可能有数十甚至数百个文件，但 Claude 只需要正确的信息就能有效地帮助你。过多不相关的上下文实际上会降低 Claude 的表现，因此学会引导它关注相关文件和文档是一项必备技能。

## /init 命令

当你首次在新项目中启动 Claude 时，运行 `/init` 命令。这会让 Claude 分析你的整个代码库并理解：

- 项目的目的和架构
- 重要的命令和关键文件
- 代码模式和结构

分析完成后，Claude 会创建一个摘要并将其写入 CLAUDE.md 文件。当 Claude 请求创建此文件的权限时，你可以按 Enter 键批准每个写入操作，或按 Shift+Tab 让 Claude 在整个会话中自由写入文件。

## CLAUDE.md 文件

CLAUDE.md 文件有两个主要用途：

1. **引导 Claude 浏览代码库**，指出重要的命令、架构和编码风格
2. **允许你给 Claude 特定或自定义的指令**

这个文件会被包含在你向 Claude 发起的每个请求中，因此它就像为你的项目设置了一个持久的系统提示词。

### CLAUDE.md 文件位置

Claude 会在三个常见位置识别三种不同的 CLAUDE.md 文件：

| 文件 | 位置 | 用途 |
|------|------|------|
| CLAUDE.md | 项目根目录 | 通过 `/init` 生成，提交到版本控制，与其他工程师共享 |
| CLAUDE.local.md | 项目根目录 | 不与其他工程师共享，包含个人指令和自定义设置 |
| ~/.claude/CLAUDE.md | 用户主目录 | 用于你机器上的所有项目，包含你希望 Claude 在所有项目中遵循的指令 |

## 添加自定义指令

你可以通过向 CLAUDE.md 文件添加指令来自定义 Claude 的行为。例如，如果 Claude 在代码中添加太多注释，你可以通过更新文件来解决这个问题。

使用 `#` 命令进入"记忆模式"——这让你智能地编辑 CLAUDE.md 文件。只需输入类似这样的内容：

```
# 谨慎使用注释。只在复杂的代码处添加注释。
```

Claude 会自动将此指令合并到你的 CLAUDE.md 文件中。

## 使用 @ 符号引用文件

当你需要 Claude 查看特定文件时，使用 `@` 符号后跟文件路径。这会自动将该文件的内容包含在你的请求中。

例如，如果你想询问关于认证系统的问题，并且你知道相关文件，你可以输入：

```
认证系统是如何工作的？@auth
```

Claude 会显示一个与 auth 相关的文件列表供你选择，然后将选中的文件包含在你的对话中。

## 在 CLAUDE.md 中引用文件

你也可以在 CLAUDE.md 文件中直接使用相同的 `@` 语法引用文件。这对于与项目许多方面相关的文件特别有用。

例如，如果你有一个定义数据结构的数据库模式文件，你可以在 CLAUDE.md 中添加：

```
数据库模式定义在 @prisma/schema.prisma 文件中。任何时候你需要了解存储在数据库中的数据结构，都可以参考它。
```

当你以这种方式引用文件时，其内容会自动包含在每个请求中，因此 Claude 可以立即回答关于你的数据结构的问题，而不必每次都搜索和读取模式文件。

## 最佳实践总结

1. **首次使用 `/init`**：在新项目中运行 `/init` 让 Claude 快速了解项目结构
2. **分层管理指令**：使用 CLAUDE.md（共享）、CLAUDE.local.md（个人）、~/.claude/CLAUDE.md（全局）三级配置
3. **善用 `@` 引用**：在对话中使用 `@文件名` 快速引用相关文件
4. **在 CLAUDE.md 中预引用**：将常用的核心文件引用添加到 CLAUDE.md，让 Claude 始终了解关键代码
5. **使用 `#` 命令**：快速添加或修改 CLAUDE.md 中的指令
