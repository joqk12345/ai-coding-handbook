# 2.8: 对话管理与上下文控制

在使用 Claude 处理复杂任务时，你通常需要引导对话以保持其专注和高效。有几种技巧可以用来控制对话流程，帮助 Claude 保持正确的方向。

## 使用 Escape 键中断 Claude

有时 Claude 会开始走向错误的方向，或者试图一次性处理太多内容。你可以按 **Escape** 键来停止 Claude 的当前回复，从而让你可以重新引导对话。

这在希望 Claude 专注于特定任务而不是同时处理多个任务时特别有用。例如，如果你要求 Claude 为多个函数编写测试，而它开始为所有函数创建综合计划，你可以中断它并要求它一次只专注于一个函数。

### 结合 Memory 使用 Escape 键

Escape 技巧最强大的应用之一是修复重复性错误。当 Claude 在不同对话中重复犯同样的错误时，你可以：

1. 按 Escape 停止当前回复
2. 使用 `#` 快捷键添加关于正确方法的记忆
3. 使用更正后的信息继续对话

这可以防止 Claude 在你项目的未来对话中犯同样的错误。

## 回溯对话

在长时间对话中，你可能会积累变得不相关或分散注意力的上下文。例如，如果 Claude 遇到错误并花时间调试它，那些来回讨论可能对下一个任务没有用处。

你可以通过**按 Escape 键两次**来回溯对话。这会显示你发送的所有消息，允许你跳回到较早的位置并从那里继续。这种技巧帮助你：

- 保持有价值的上下文（如 Claude 对你的代码库的理解）
- 移除分散注意力或不相关的对话历史
- 让 Claude 专注于当前任务

## 上下文管理命令

Claude 提供了几个命令来帮助有效管理对话上下文：

### /compact

`/compact` 命令会总结你的整个对话历史，同时保留 Claude 学到的关键信息。这在以下情况下非常理想：

- Claude 已经获得了关于你项目的宝贵知识
- 你想继续处理相关任务
- 对话已经变长但包含重要上下文

当 Claude 已经学到了很多关于当前任务的知识，而你希望它在转移到下一个相关任务时保持这些知识时，使用 `compact`。

### /clear

`/clear` 命令会完全删除对话历史，给你一个全新的开始。这在以下情况下最有用：

- 你正在切换到一个完全不同的、不相关的任务
- 当前对话上下文可能会让 Claude 对新任务感到困惑
- 你想在没有任何先前上下文的情况下重新开始

## 何时使用这些技巧

这些对话控制技巧在以下情况特别有价值：

- **长时间运行的对话**，上下文可能变得混乱
- **任务转换**，先前的上下文可能会分散注意力
- **Claude 重复犯同样错误**的情况
- **复杂项目**，你需要将注意力集中在特定组件上

通过策略性地使用 Escape、双击 Escape、`/compact` 和 `/clear`，你可以让 Claude 在整个开发工作流程中保持专注和高效。这些不仅仅是便利功能——它们是维持有效 AI 辅助开发会话的基本工具。
