# 2.10: MCP サーバー拡張

`2.1 > 2.2 > 2.3 > 2.4 > 2.5 > 2.6 > 2.7 > 2.8 > 2.9 | [ 2.10 ] 2.11`

> *"MCP は Claude をコーディングアシスタントからフルスタックパートナーへ進化させる"* -- プロトコル拡張により無限の可能性を解き放つ。

## 問題

Claude Code の即座の能力は、組み込みツールセット（Bash、Read、Write、Edit）に制限されています。以下のシナリオでは、これらのツールが力不足です：

- **Web 開発デバッグ**: ソースコードを手動で確認し、スタイル問題を推測する必要があり、レンダリング効果を直接見ることができない
- **データベース操作**: SQL クエリスクリプトを書く必要があり、データを直接対話的に探索できない
- **API テスト**: 外部ツール（curl、Postman）が必要で、コンテキストを頻繁に切り替える必要がある
- **ブラウザ自動化**: Selenium/Playwright スクリプトを書く必要があり、障壁が高く、デバッグが困難

本質的に、Claude と外部世界の相互作用は「ファイル+コマンド」の抽象層に制限されています。 MCP（Model Context Protocol）の登場により、Claude に標準化された能力拡張プロトコルが提供されました。

## 解決策

```
+-------------------+      +-------------------+      +-------------------+
|   Claude Code      |      |   MCP Server      |      |   External        |
|   (Host)          |<---->|   (Playwright)   |<---->|   Resources      |
|                   |      |                   |      |   - Browser        |
|  Tool: mcp__*     |      |  Tools:           |      |   - Database       |
|  - browser_goto   |      |  - browser_goto   |      |   - API endpoints  |
|  - browser_click  |      |  - browser_click  |      |   - Files          |
|  - browser_type   |      |  - browser_type   |      |                   |
+-------------------+      +-------------------+      +-------------------+
         ^
         |
    標準化プロトコル (MCP)
    - ツール発見
    - パラメータ検証
    - 結果返却
```

MCP の中核的価値：

| 特性 | 説明 | 利点 |
|------|------|------|
| プロトコル標準化 | 統一されたサービス発見、呼び出し、返却フォーマット | 一度接続すれば、どこでも利用可能 |
| 能力拡張性 | サーバーの追加により新しい能力をアンロック | 「何ができるか」から「何をしたいか」へ |
| 安全制御 | 各ツールに個別に承認、事前承認可能 | エンタープライズグレードのセキュリティコンプライアンス |
| エコシステム共有 | コミュニティが貢献したサーバーを再利用可能 | 車輪の再発明を避ける |

## 仕組み

### 1. MCP アーキテクチャ概要

MCP はクライアント-サーバーアーキテクチャを採用しています：

```
┌─────────────────────────────────────────────────────────┐
│                      Host (Claude Code)                  │
│  ┌─────────────────────────────────────────────────┐     │
│  │              MCP Client (ビルトイン)             │     │
│  │  - サービス発見 (Tools/List)                     │     │
│  │  - ツール呼び出し (Tools/Call)                   │     │
│  │  - セッション管理                              │     │
│  └────────────────┬────────────────────────────────┘     │
│                   │ MCP Protocol (stdio/sse)             │
└───────────────────┼──────────────────────────────────────┘
                    │
        ┌───────────┴───────────┐
        │                       │
   ┌────▼─────┐           ┌──────▼─────┐
   │ Playwright│           │  Database  │
   │ Server    │           │  Server    │
   └───────────┘           └────────────┘
```

### 2. サーバーインストールと設定

Playwright MCP サーバーを例に：

**ステップ 1: サーバーインストール**

```bash
# ターミナルで実行（Claude Code 内部ではない）
claude mcp add playwright npx @playwright/mcp@latest
```

このコマンドは：
1. MCP サーバーに "playwright" という名前を付ける
2. サーバー起動コマンドを指定する

**ステップ 2: 権限設定（オプションだが推奨）**

MCP サーバーツールを初めて使用する際、Claude は権限を求めます。設定ファイルで事前承認が可能です：

```json
// .claude/settings.local.json
{
  "permissions": {
    "allow": ["mcp__playwright"],
    "deny": []
  }
}
```

**ステップ 3: Claude Code の再起動**

設定変更後、新しいコマンドを読み込むために Claude Code を再起動する必要があります。

### 3. ツール呼び出しフロー

Claude が MCP ツールを使用する必要がある場合、以下のフローに従います：

```python
# 1. ツール発見（起動時またはオンデマンド）
available_tools = mcp_client.list_tools(server="playwright")
# 返却: [browser_goto, browser_click, browser_type, browser_screenshot, ...]

# 2. ツール呼び出し
result = mcp_client.call_tool(
    server="playwright",
    tool="browser_goto",
    arguments={"url": "http://localhost:3000"}
)

# 3. 結果処理
if result.success:
    # 結果を会話コンテキストに注入
    messages.append({
        "role": "user",
        "content": f"ブラウザページコンテンツ: {result.content}"
    })
else:
    # エラー処理
    messages.append({
        "role": "user",
        "content": f"ツール呼び出しエラー: {result.error}"
    })
```

## 他の MCP サーバーの探索

Playwright は MCP サーバーの可能性の一例に過ぎません。エコシステムには以下の機能のためのサーバーが含まれています：

- **データベースインタラクション** - データを直接クエリして可視化
- **API テストと監視** - 自動化された API 検証
- **ファイルシステム操作** - 高度なファイル管理
- **クラウドサービス統合** - AWS、GCP、Azure と相互作用
- **開発ツール自動化** - IDE、CI/CD と統合

特定の開発ニーズに合わせた MCP サーバーを探索することを検討してください。それらは Claude をコードアシスタントから、ツールチェーン全体と相互作用できる包括的な開発パートナーに変えることができます。

## 変更点の比較

| コンポーネント | 以前 | 以降 (2.10) |
|-------------|------|-------------|
| ツールセット | 4 つの組み込み基本ツール | MCP による無限の拡張 |
| Web デバッグ | 手動でソースを確認して推測 | ブラウザでの視覚的検査 |
| データベース操作 | SQL スクリプトを書く | MCP で直接相互作用 |
| API テスト | 外部ツールの切り替え | Claude 内で完了 |
| 能力境界 | 固定 | 動的に拡張可能 |

## ベストプラクティス

### MCP サーバー選択ガイド

| シナリオ | 推奨 MCP サーバー | 重要ツール |
|---------|-----------------|------------|
| Web 開発 | Playwright | browser_goto, screenshot, click |
| データベース | SQLite/PostgreSQL | query, schema, visualize |
| API 開発 | HTTP/REST | request, mock, validate |
| クラウド開発 | AWS/GCP/Azure | deploy, monitor, configure |

### 安全な使用に関する推奨事項

1. **権限の最小化**: 必要なサーバーのみを事前承認する
   ```json
   {
     "permissions": {
       "allow": ["mcp__playwright"],
       "deny": ["mcp__filesystem_delete"]
     }
   }
   ```

2. **サーバー出所の確認**: 公式またはコミュニティ検証済みのサーバーを優先する

3. **機密操作の確認**: 本番環境に関与する操作は手動確認ステップを保持する

4. **定期的な監査**: `.claude/settings.local.json` の権限設定を定期的に確認する

MCP サーバーを適切に設定して使用することで、Claude Code を強力なコードアシスタントから、開発ツールチェーン全体と深く統合された包括的な開発パートナーにアップグレードできます。
