# 第25章：Autoresearch 实测提速 11% —— Agent 自主调参正在成为“最终战场”

> 三天前，Karpathy 将 autoresearch 挂在 nanochat 的 depth=12 模型上连续跑了约两天。结果找到约 20 项可叠加改动，全部迁移到 depth=24 后依然成立，最终把榜单指标 **Time to GPT-2** 从 **2.02 小时**降到 **1.80 小时**（约 11% 提升）。

## 1. 这次实验到底发生了什么？

这次“round 1”实验并不是一次单点优化，而是一个完整的自动化研究闭环：

1. Agent 自主发起并运行大量实验（约 700 次改动尝试）；
2. 根据验证集损失（validation loss）结果，自动规划下一批实验；
3. 汇总可叠加、可迁移的改动并放大到更大模型；
4. 以统一指标（Time to GPT-2）评估最终收益。

关键点不在于“某一条神奇技巧”，而在于：**多个真实有效的小改动被系统化发现并稳定叠加**。

## 2. 关键收益：从“能优化”到“端到端自动优化”

Karpathy 特别强调，这些改动并非噪声：

- 在小模型上有效；
- 迁移到更大模型后仍有效；
- 叠加后带来可量化收益（2.02h → 1.80h）；
- 且其中不少点是人工长期调参中未被发现的“盲区”。

这也是很多工程团队关注的核心：**Agent 不只是执行指令，而是在做完整的实验—反馈—再规划循环**。

## 3. Agent 找到的代表性改动（摘录）

在公开描述中，较有代表性的改动包括：

- **QKnorm 缩放疏漏修复**：无参 QKnorm 缺少 scaler multiplier，导致注意力过于弥散；
- **Value Embeddings 正则化补齐**：此前几乎未加，补齐后效果明显；
- **Banded Attention 重新调参**：原设置过于保守；
- **AdamW betas 修正**：超参数配置存在问题；
- **Weight Decay 调度优化**：训练过程中衰减策略更合理；
- **网络初始化（Initialization）优化**：改善早期训练动态。

这些都属于“并不花哨但非常工程化”的改进类型：可复现、可组合、可迁移。

## 4. 为什么说这是“最终战场”？

原文给出的判断非常直接：**所有前沿实验室都会做这件事**。

原因是它符合一个朴素而强大的工程逻辑：

- 只要某个目标指标可评估（或能找到可近似的代理指标），
- 就可以把它包装成自动化搜索与优化问题，
- 通过 agent swarm（多智能体群体）持续并行探索。

在大规模场景里，复杂度会显著上升（不再是单一训练脚本），但方向不变：

- 用小规模实验筛选想法；
- 将高潜改动逐步提升到大规模训练；
- 人类更多参与边界定义、约束设计与策略决策。

## 5. 给工程团队的落地启示

如果你的问题满足以下条件，就很适合做 autoresearch：

1. 有明确优化目标（延迟、损失、通过率、成本、稳定性等）；
2. 目标可以高频评估，或能构建低成本代理任务；
3. 改动空间可参数化（配置、提示、流程、架构、策略）；
4. 能建立自动回归与安全护栏。

换句话说：**“可测量” + “可迭代” 的问题，几乎都能被 agent swarm 系统性优化。**
