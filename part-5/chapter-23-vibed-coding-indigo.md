# 第23章：周末氛围直播收官总结——软件未死，只是正在被 Agent 重写

> *上周末氛围直播的最后一个小总结：从 2025 年底开始，越来越多软件开发者已经转向“为 AI Agent 开发工具”，而不是只为人类界面写软件。*

## 引言：资本市场喊“软件已死”，但它已经“死过很多次”

“软件已死”不是第一次出现的叙事。每一轮技术变革里，都会有人宣告旧软件范式终结。

但历史更像是：**旧形态退场，新形态接管**。

- MS-DOS 时代：软件通过软盘和光盘分发
- 互联网时代：共享软件与在线下载兴起
- 云计算时代：SaaS 成为主流，大家共用逻辑层，工作去适配软件
- Agent 时代：软件开始动态生成，反过来适配具体工作流

所以与其说“软件死了”，不如说：

> *静态软件正在退场，动态软件正在崛起。*

---

## 第一部分：从“人适应软件”到“软件适应工作”

SaaS 时代的核心收益是规模化：
- 一套逻辑服务很多客户
- 通过标准化换取成本效率
- 用户必须在流程上做妥协

AI Agent 时代的核心收益是个性化：
- 同样的业务目标，不同行业/团队可自动生成不同实现
- 软件不再只有“版本”，而是持续变化的“过程”
- 工作流成为第一性，软件成为可塑的执行层

这就是范式切换：

> *过去是“工作适配软件”，现在是“软件适配工作”。*

---

## 第二部分：为什么岗位没有消失，反而更活跃？

很多人担心 Agent 会替代工程师，但现实是另一条曲线：**工具门槛降低后，需求总量上升**。

过去只有世界 500 强有预算雇豪华团队定制系统；现在，一两个工程师配合几十个 Agent，就能完成过去需要完整部门才能推进的项目。

这带来两个直接结果：

1. **老板更敢投**：因为试错成本显著下降，立项速度更快。
2. **岗位更细分**：工程师不再只是“写代码的人”，而是“编排 Agent、定义上下文、验收结果的人”。

换句话说，Agent 不是简单替代人，而是放大了每个工程师的产出半径，进而释放更多需求与职位。

---

## 第三部分：2025 年底之后，开发者为何转向“Agent 工具开发”

从 2025 年底开始，一个明显趋势是：

- 开发者不再满足于做单点应用
- 开始搭建 Skills、Workflow、Tooling、Context Infrastructure
- “给 Agent 用的工具”正在成为新的产品层

因为在新栈里，竞争力不只来自 UI，而来自：
- 是否有高质量工具链可调用
- 是否有可复用任务模板
- 是否有持续更新的业务上下文系统

这就是为什么“AI Native 软件”看起来像软件，但内核更接近“可编排能力网络”。

---

## 第四部分：模型自我进化的新判断——Context 即学习现场

Anthropic CEO Dario 在 2025 年底采访中的判断引发了广泛讨论：

> *只要给模型足够巨大、并能随业务动态升级的上下文（Context），模型就能在专属 Context 中完成针对性学习并立即投入使用。*

这意味着一种可能：

- 许多过去必须依赖“持续微调训练”的场景
- 未来可改由“持续更新上下文”来实现快速进化

简化理解：
- 训练像“改造大脑”
- Context 像“实时工作记忆 + 外接知识系统”

当 Context 足够精准、足够新鲜、足够可执行时，模型表现会呈现出“近似持续学习”的效果。

---

## 第五部分：为什么硬件必须跟着变？

如果软件的核心竞争从“功能列表”转向“超大规模、动态上下文处理能力”，那么底层硬件迟早要重构。

关键压力点包括：

- **内存容量与带宽**：需要容纳更长、更复杂的上下文窗口
- **存储与检索链路**：需要低延迟地组织企业级知识与历史任务轨迹
- **互连与传输架构（含光传导方向）**：需要支撑更高密度的数据流动与推理协同

所以这不是单纯的模型竞赛，而是“模型 + 数据 + 系统 + 硬件”的全栈升级。

---

## 第六部分：给工程师的行动建议

1. **从写功能，转向造能力**：多做可复用工具、技能模板和上下文资产。
2. **把上下文工程当主战场**：需求文档、业务规则、验收标准都要结构化。
3. **拥抱多 Agent 协作**：把自己升级为任务设计者与质量守门人。
4. **关注基础设施变化**：模型迭代很快，但真正形成壁垒的是系统化落地能力。

---

## 总结：软件没有死，正在进入“动态生成时代”

资本市场口中的“软件已死”，本质上是在说：

- 过去那种静态交付、低频迭代的软件形态正在失效
- 新的软件将由 Agent 按业务实时生成并持续演进

对工程师来说，这不是终局，而是新机会：

> *一两个工程师 + 数十个 Agent，已经可以形成过去无法想象的生产力。*

接下来真正拉开差距的，不是“会不会用 AI”，而是：

- 你是否能构建高质量上下文系统
- 你是否能设计可扩展的 Agent 工作流
- 你是否能把业务问题稳定地转译成机器可执行的能力

软件未死，它只是从“产品”变成了“过程”。
