# 贡献指南

感谢你参与《AI-Coding 使用手册》共建！

## 文件命名规则（已定）

本仓库采用**稳定 slug** 方案（推荐）：

- 章节文件名使用语义化 slug（如 `chapter-4-11-github-integration.md`）。
- **文件名不代表最终章节号**，不要因为章节前后顺序调整而频繁重命名文件。
- 章节编号以以下两处为准：
  - `SUMMARY.md` 的目录顺序与展示文案
  - 章节正文的 H1 标题

> 结论：调整章节顺序时，优先改目录与标题，不改文件名。

## 章节变更时请同步

请参考 `docs/structure.md` 的完整流程与检查项，至少确保以下三处一致：

1. `SUMMARY.md`（目录顺序/显示编号）
2. 章节正文 H1（显示编号）
3. 可视化文案（`visualizations/`）

## 提交流程建议

1. 修改内容后，检查目录、标题、可视化三处是否同步。
2. 如有新增章节，在 `SUMMARY.md` 与 `.vitepress/config.mts` 同步新增入口。
3. 提交 PR 时在描述中注明：
   - 本次是否涉及章节重排
   - 三处同步点是否已检查
