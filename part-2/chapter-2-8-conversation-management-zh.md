# 2.8: 对话管理与上下文控制

`2.1 > 2.2 > 2.3 > 2.4 > 2.5 > 2.6 > 2.7 | [ 2.8 ] 2.9 > 2.10 > 2.11`

> *"掌控对话就是掌控结果"* -- 精准的上下文管理让 AI 始终保持专注。

## 问题

在使用 Claude 处理复杂任务时，长对话容易偏离主题。上下文累积会导致：
- 模型注意力分散，输出质量下降
- 错误信息在对话中反复传播
- 任务切换时上下文混杂
- 调试信息污染后续任务

没有有效的对话控制机制，用户只能被动接受模型的输出方向。

## 解决方案

```
+----------------+      +----------------+      +----------------+
|   用户输入      | ---> |   对话管理      | ---> |   模型响应      |
|   (Prompt)     |      |   (控制中枢)    |      |   (Response)   |
+----------------+      +--------+-------+      +----------------+
                               |
                    +----------+-----------+
                    |                      |
               +----v----+           +-----v-----+
               | Escape  |           | /compact |
               |  中断   |           | /clear   |
               +---------+           +-----------+
```

三种核心控制机制：

| 机制 | 触发方式 | 作用 | 适用场景 |
|------|---------|------|---------|
| Escape 中断 | 按 Escape 键 | 停止当前生成 | 模型偏离方向时 |
| 对话回溯 | 双击 Escape | 跳转到历史位置 | 移除无效上下文 |
| 命令控制 | `/compact` `/clear` | 管理对话状态 | 任务切换时 |

## 工作原理

### 1. Escape 中断机制

当模型开始生成不相关或错误内容时，立即按 **Escape** 键：

```python
# 伪代码示意
if user_pressed_escape:
    stop_generation()
    preserve_context_until_interruption()
    wait_for_new_prompt()
```

**使用场景：**
- 模型试图一次性处理太多任务
- 生成内容与预期不符
- 发现模型开始重复之前的错误

### 2. 结合 Memory 修复重复错误

当 Claude 在不同对话中重复犯同样错误时：

```
步骤 1: 按 Escape 停止当前错误输出
步骤 2: 使用 # 添加快捷记忆 (Memory)
        格式: # 正确做法: [正确的规则]
步骤 3: 用更正后的信息继续对话
```

这会将正确规则写入长期记忆，防止未来对话中再次出现相同错误。

### 3. 对话回溯（双击 Escape）

```
按 Escape 一次: 停止当前生成
按 Escape 两次: 显示对话历史导航

+-------------------------------+
|  对话回溯界面                  |
+-------------------------------+
|  [1] 用户: 创建测试文件        |
|  [2] Claude: 创建了 test.py   |
|  [3] 用户: 运行测试           | <-- 跳转到此处
|  [4] Claude: 错误信息...      |
|  [5] 用户: 修复这个bug         |
+-------------------------------+
```

**回溯的作用：**
- 保留有价值的上下文（对代码库的理解）
- 移除分散注意力的调试历史
- 让 Claude 专注于当前任务

### 4. 上下文管理命令

#### `/compact` 命令

```python
# 执行前: 长对话历史
messages = [
    {"role": "user", "content": "创建项目"},
    {"role": "assistant", "content": "已创建..."},
    # ... 50+ 条消息
    {"role": "user", "content": "运行测试"},
]

# compact 操作
summary = generate_summary(messages)  # 提取关键信息
messages = [
    {"role": "system", "content": f"对话摘要: {summary}"},
    {"role": "user", "content": "运行测试"},
]
```

**适用场景：**
- Claude 已获得宝贵的项目知识
- 想继续相关任务但对话过长
- 需要保留上下文但减少 token 消耗

#### `/clear` 命令

```python
# 执行 clear 后
messages = []  # 完全清空
```

**适用场景：**
- 切换到完全不同的任务
- 当前上下文可能让 Claude 对新任务困惑
- 想完全重新开始

## 变更对比

| 组件 | 之前 | 之后 (2.8) |
|------|------|-----------|
| 对话控制 | 无中断机制 | Escape 键中断 |
| 历史导航 | 无 | 双击 Escape 回溯 |
| 上下文管理 | 手动重启 | `/compact` 总结保留 |
| 状态重置 | 关闭重开 | `/clear` 快速清空 |
| 错误修复 | 反复提示 | 结合 Memory 永久修正 |

## 实践建议

### 何时使用这些技巧

| 场景 | 推荐操作 |
|------|---------|
| 模型偏离方向 | 按 Escape 重新引导 |
| Claude 重复犯错 | Escape + 添加 Memory |
| 调试后切换任务 | 双击 Escape 回溯到干净状态 |
| 长对话继续相关任务 | `/compact` |
| 完全不同的新任务 | `/clear` |

### 高效对话管理流程

```
1. 开始任务
   ↓
2. 发现模型偏离 → Escape 中断
   ↓
3. 必要时添加 Memory 防止重复错误
   ↓
4. 任务完成或需要切换
   ↓
5. 判断:
      - 继续相关任务 → /compact
      - 新任务且无价值上下文 → /clear
      - 移除调试历史 → 双击 Escape 回溯
```

通过策略性地使用 Escape、双击 Escape、`/compact` 和 `/clear`，你可以让 Claude 在整个开发工作流程中保持专注和高效。这些不仅仅是便利功能——它们是维持有效 AI 辅助开发会话的基本工具。
