# 2.8: 会話管理とコンテキスト制御

`2.1 > 2.2 > 2.3 > 2.4 > 2.5 > 2.6 > 2.7 | [ 2.8 ] 2.9 > 2.10 > 2.11`

> *"会話を制御することは結果を制御すること"* -- 正確なコンテキスト管理が AI の集中力と効率性を保つ。

## 問題

Claude を複雑なタスクに使用する際、長い会話は話題をずれやすくなります。コンテキストの蓄積により以下が発生します：
- モデルの注意が散漫になり、出力品質が低下
- 誤った情報が会話全体に繰り返し広がる
- タスク切り替え時にコンテキストが混在
- デバッグ情報が後続タスクを汚染

効果的な会話制御メカニズムがない場合、ユーザーはモデルの出力方向を受動的に受け入れるしかありません。

## 解決策

```
+----------------+      +----------------+      +----------------+
|   ユーザー入力  | ---> |   会話管理      | ---> |   モデル出力    |
|   (Prompt)     |      |   (制御中枢)    |      |   (Response)   |
+----------------+      +--------+-------+      +----------------+
                               |
                    +----------+-----------+
                    |                      |
               +----v----+           +-----v-----+
               | Escape  |           | /compact |
               |  中断   |           | /clear    |
               +---------+           +-----------+
```

3つのコア制御メカニズム：

| メカニズム | トリガー | 効果 | 使用場面 |
|-----------|---------|------|----------|
| Escape 中断 | Escape キー押下 | 現在の生成を停止 | モデルがずれた時 |
| 会話巻き戻し | Escape ダブルタップ | 履歴位置にジャンプ | 無効なコンテキストを除去 |
| コマンド制御 | `/compact` `/clear` | 会話状態を管理 | タスク切り替え時 |

## 仕組み

### 1. Escape 中断メカニズム

モデルが無関係または誤ったコンテンツの生成を開始した場合、すぐに **Escape** を押します：

```python
# 擬似コードでの説明
if user_pressed_escape:
    stop_generation()
    preserve_context_until_interruption()
    wait_for_new_prompt()
```

**使用場面：**
- モデルが一度に多くのタスクを処理しようとする
- 生成されたコンテンツが期待と一致しない
- モデルが以前のエラーを繰り返し始める

### 2. Memory と組み合わせた繰り返しエラーの修正

Claude が異なる会話間で同じ間違いを繰り返す場合：

```
ステップ 1: Escape を押して現在の誤った出力を停止
ステップ 2: # を使用してクイックメモリを追加
        形式: # 正しいアプローチ: [正しいルール]
ステップ 3: 修正された情報で会話を継続
```

これにより正しいルールが長期記憶に書き込まれ、将来の会話で同じエラーが防止されます。

### 3. 会話巻き戻し（Escape ダブルタップ）

```
Escape を1回押す: 現在の生成を停止
Escape を2回押す: 会話履歴ナビゲーションを表示

+-------------------------------+
|  会話巻き戻し                  |
+-------------------------------+
|  [1] ユーザー: テストファイル作成 |
|  [2] Claude: test.py を作成    |
|  [3] ユーザー: テスト実行      <-- ここにジャンプ
|  [4] Claude: エラーメッセージ... |
|  [5] ユーザー: このバグを修正  |
+-------------------------------+
```

**巻き戻しの利点：**
- 価値あるコンテキストを保持（コードベースの理解）
- 気を散らすデバッグ履歴を除去
- Claude を現在のタスクに集中させる

### 4. コンテキスト管理コマンド

#### `/compact` コマンド

```python
# 実行前: 長い会話履歴
messages = [
    {"role": "user", "content": "プロジェクトを作成"},
    {"role": "assistant", "content": "作成しました..."},
    # ... 50+ メッセージ
    {"role": "user", "content": "テストを実行"},
]

# compact 操作
summary = generate_summary(messages)  # 重要情報を抽出
messages = [
    {"role": "system", "content": f"会話の要約: {summary}"},
    {"role": "user", "content": "テストを実行"},
]
```

**使用場面：**
- Claude が価値あるプロジェクト知識を獲得した
- 関連タスクを継続したいが会話が長すぎる
- コンテキストを保持しつつ token 消費を減らしたい

#### `/clear` コマンド

```python
# clear 実行後
messages = []  # 完全にクリア
```

**使用場面：**
- 完全に異なるタスクに切り替える
- 現在のコンテキストが新しいタスクで Claude を混乱させる可能性がある
- 完全に新しく始めたい

## 変更点の比較

| コンポーネント | 以前 | 以降 (2.8) |
|-------------|------|-----------|
| 会話制御 | 中断メカニズムなし | Escape キー中断 |
| 履歴ナビゲーション | なし | Escape ダブルタップで巻き戻し |
| コンテキスト管理 | 手動で再起動 | `/compact` で要約保持 |
| 状態リセット | 閉じて再開 | `/clear` でクリア |
| エラー修正 | 繰り返しプロンプト | Memory と組み合わせた永久修正 |

## 実践的なアドバイス

### いつこれらのテクニックを使うか

| シナリオ | 推奨アクション |
|---------|--------------|
| モデルがずれる | Escape を押して方向修正 |
| Claude が間違いを繰り返す | Escape + Memory 追加 |
| デバッグ後にタスク切り替え | Escape ダブルタップで巻き戻し |
| 長い会話で関連タスクを継続 | `/compact` |
| 完全に新しい無関連タスク | `/clear` |

### 効率的な会話管理フロー

```
1. タスク開始
   ↓
2. モデルのずれを検出 → Escape 中断
   ↓
3. 必要に応じて Memory を追加して繰り返しエラーを防止
   ↓
4. タスク完了または切り替えが必要
   ↓
5. 判断:
      - 関連タスクを継続 → /compact
      - 価値あるコンテキストなしの新タスク → /clear
      - デバッグ履歴を除去 → Escape ダブルタップで巻き戻し
```

Escape、Escape ダブルタップ、`/compact`、`/clear` を戦略的に使用することで、Claude を開発ワークフロー全体で集中させ効率的に保つことができます。これらは単なる便利機能ではなく、効果的な AI 支援開発セッションを維持するための基本ツールです。
