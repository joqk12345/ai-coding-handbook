# Agent 本质

## 第一部分：先理解 Agent 文件（AGENTS.md）

参考 [agents.md](https://agents.md/) 的约定，**AGENTS.md 可以理解为“给 AI Agent 的项目内操作手册”**：

- 它通常放在仓库根目录或子目录中，用来声明该目录下的工作规则。
- 常见内容包括：代码风格、目录约定、测试与提交流程、PR 规范等。
- Agent 在执行任务时会优先读取这些规则，并按目录作用域继承；更深层目录的 AGENTS.md 会覆盖上层同名规则。

这件事很关键，因为它决定了 Agent 不是“会写代码的聊天机器人”，而是一个**可被约束、可被协作流程接入、可复用组织经验**的执行单元。换句话说，Agent 的能力不只来自模型本身，也来自项目给它的“制度化上下文”。

1. 回归 Agent 本质 - 规划、记忆、工具调用  
   以规划（Planning）、记忆（Memory）、工具调用（Tool Use）为核心能力，构建可执行、可持续优化的智能体。

2. 加速迭代效率 - Headless Mode、Agent SDK、评估生态  
   通过 Headless Mode、Agent SDK 与评估生态（Eval）打通开发-测试-上线闭环，把试错成本降到最低。

3. 重新定义生产关系 - 技术选型简单、AI 迭代 AI  
   让技术选型更轻、更快，形成“AI 迭代 AI”的新范式，人类负责目标与约束，AI 负责实现与优化。
