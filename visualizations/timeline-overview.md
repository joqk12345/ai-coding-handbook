# 学习路径时间线

本页基于当前手册全部章节，给出从认知建立到工程自治的完整学习时间线。

## 全书时间线（按能力跃迁）

| 阶段 | 章节范围 | 能力目标 | 建议产出 |
|---|---|---|---|
| 0. 认知校准 | 第1-2章、第4章、Agent 本质 | 建立 AI 编程心智模型，理解”助手”与”Agent”的边界 | 一份个人使用边界清单（适用/不适用场景） |
| 1. 工具掌握 | 第2章 Claude 手册、2.1-2.14 | 熟悉 Claude Code 的命令、上下文、技能、GitHub 集成与高级功能 | 一套可复用的 `.claude/` 最小配置 |
| 2. 多工具横向扩展 | 第3章 Codex、第4章 Gemini、第5章 Opencode、第6章工具概览 | 建立多模型/多工具选型能力与迁移能力 | 一张“任务类型 → 推荐工具”映射表 |
| 3. 工程化提效 | 第7章 Prompt、第8章测试调试、第9章综合实战 | 将提示工程与测试、调试流程整合到开发闭环 | 一个端到端小项目（含测试和回归） |
| 4. Agent 系统构建 | 第12-16章（构建、模式、多智能体、工作流、框架） | 从单 Agent 到任务系统、后台执行、多智能体协作 | 一套可运行的 Agent 原型（含任务/后台/协作） |
| 5. 组织级演进 | 第17-18章（怀疑者实践、实战复盘）、第19-22章（自主代码库、Harness Engineering、上下文工程） | 从个人提效升级到团队级软件交付能力 | 一份团队落地路线图（治理、触发器、扩展策略） |
| 6. 参考与持续更新 | 附录A、附录B | 建立工具与资源的持续学习机制 | 个人知识库与更新节奏 |

## 关键转折点

1. 从“会用工具”到“会设计工作流”：发生在 2.3（Skills）与 2.6（上下文管理）。
2. 从“单点提效”到“可验证交付”：发生在第8章（测试调试）与第9章（综合实战）。
3. 从”单 Agent”到”系统能力”：发生在第12-14章（构建、模式、多智能体）。
4. 从”个人效率”到”组织速度”：发生在第17-18章（怀疑者实践、实战复盘）与第19-22章（自主代码库、Harness Engineering、上下文工程）。

## 学习路径建议

1. 新手路径（4-6周）：阶段 0 → 1 → 3。
2. 进阶路径（6-8周）：阶段 0 → 1 → 2 → 3 → 4。
3. 架构/平台路径（持续）：阶段 1 → 4 → 5，并配合附录做长期更新。
